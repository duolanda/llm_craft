# Sprint - 当前问题清单

> 记录 MVP 测试中发现的问题和待改进项

## 高优先级

### 1. Token 消耗过快
- **描述**: AI 每 5 ticks (2.5 秒) 调用一次 API，3 分钟消耗 50w+ tokens
- **影响**: 成本极高，无法长时间运行
- **可能的解决方案**:
  - 降低 AI 调用频率 (如每 10-20 ticks 一次)
  - 使用缓存，相同状态下复用上次生成的代码
  - 支持使用更便宜的模型 (如 gpt-3.5-turbo 或本地小模型)
  - 添加 Token 消耗统计和上限限制
  - 允许用户手动触发 AI 思考 (而非自动)

### 2. 单位重叠问题 ✅ 已完成
- **描述**: 多个单位可以占据同一个格子，缺乏碰撞检测
- **影响**: 游戏真实性降低，单位堆叠导致视觉混乱
- **可能的解决方案**:
  - 添加单位碰撞检测，禁止移动到已有单位的格子
  - 或者允许重叠但视觉上错开显示

### 3. 支持双方使用不同 API/模型
- **描述**: 目前双方 AI 使用相同的 API Key 和模型
- **影响**: 无法对比不同模型的策略差异
- **可能的解决方案**:
  - 前端添加配置面板，分别为红蓝双方设置 API Key、Base URL、模型名称
  - 后端 GameOrchestrator 支持为每个玩家配置独立的 OpenAIClient

## 中优先级

### 4. 观赏性改进 - 数据面板 ✅ 已完成
- **描述**: 无法直观看到双方单位数量、资源数量、摧毁/死亡统计
- **已完成内容**:
  - 左侧数据面板实时显示:
    - 双方单位数量 (Worker/Soldier/Scout 分别统计，带图例)
    - 当前能量值和每 tick 产能
    - 建筑数量 (HQ / 发电 / 兵营)
    - 游戏时间和 tick 数
  - 增加单位编制颜色图例，提升可读性

### 5. 观赏性改进 - 视觉反馈
- **描述**: 攻击、摧毁等动作缺乏视觉反馈
- **可能的解决方案**:
  - 攻击时显示特效或连线
  - 单位死亡时显示爆炸/消失动画
  - 建筑被摧毁时视觉提示
  - 添加战斗日志高亮

## 低优先级

### 6. 前端 UI 改进 ✅ 已完成
- **描述**: 当前前端过于简陋，纯 CSS 内联样式难以维护
- **已完成内容**:
  - 全面重设计为工业科幻终端 HUD 风格
  - 引入 `Chakra Petch` / `Rajdhani` / `JetBrains Mono` 字体
  - 玻璃拟态面板 + 四角 HUD 装饰 + 扫描线效果
  - 三栏响应式布局（数据 | 画面 | AI 终端）
  - 增加战术图例面板，解释 Canvas 中所有图案含义
  - 统一 CSS 变量管理，移除内联样式

### 7. 回放系统
- **描述**: 游戏结束后无法回放观看
- **可能的解决方案**:
  - 将游戏快照序列保存到文件
  - 添加回放控制面板 (播放/暂停/进度条/速度控制)
  - 支持从任意 tick 开始回放

### 8. 数据面板 - 建造序列显示
- **描述**: 左侧战场数据面板目前只显示静态统计，无法看到双方正在建造/生产的队列
- **影响**: 难以判断双方的战略意图和经济投入方向
- **可能的解决方案**:
  - 在 StatsPanel 的建筑设施区域增加“正在建造”列表
  - 显示每个建筑的当前生产进度条和剩余 tick
  - 区分红蓝双方队列，按Worker/Soldier/Scout分类汇总
